## Story 1.1 - Secure desktop scaffold and route shell
Story Key: story-1.1
Epic Key: epic-1
Required for Success: yes
Depends On: none

### Goal
- Electron + React + TypeScript 기반 실행 가능한 데스크탑 앱 껍데기와 기본 라우트 셸을 만든다.

### In Scope
- 루트 워크스페이스 설정, Electron main/preload 부트스트랩, React renderer 엔트리 구성
- 기본 라우트 `Chat`, `Workbench`, `Projects`, `Usage`, `Settings`와 좌측 내비게이션 셸
- `contextIsolation` 활성화와 `nodeIntegration` 비활성화 같은 보안 기본값

### Out of Scope
- 실제 채팅 UI, 저장소, 모델 호출, 상태 동기화
- 설정 화면의 세부 폼과 비즈니스 로직

### Acceptance Criteria
- 개발 모드에서 데스크탑 앱이 실행되고 기본 경로가 `Chat`으로 열린다.
- 좌측 글로벌 내비게이션에서 `Chat`, `Workbench`, `Projects`, `Usage`, `Settings`를 볼 수 있다.
- Electron 윈도우는 `contextIsolation=true`, `nodeIntegration=false`로 부팅된다.
- 각 라우트는 최소한의 placeholder 화면으로 정상 이동되며 크래시 없이 렌더링된다.

### Validation Steps
- 개발 실행 명령으로 앱을 부팅하고 기본 경로가 `Chat`인지 확인한다.
- 모든 글로벌 내비게이션 항목을 클릭해 라우트 전환이 되는지 확인한다.
- Electron 설정 단위 테스트 또는 런타임 검사로 보안 플래그를 검증한다.

### Affected Files or Modules
- `package.json`
- `apps/desktop/src/main`
- `apps/desktop/src/preload`
- `apps/desktop/src/renderer/app`
- `apps/desktop/src/renderer/routes`

### Deferred Follow-ups
- 코드 서명, 자동 업데이트, 배포 빌드 파이프라인

## Story 1.2 - Shared IPC command, query, and event contracts
Story Key: story-1.2
Epic Key: epic-1
Required for Success: yes
Depends On: story-1.1

### Goal
- main process와 renderer가 같은 명령, 조회, 이벤트 계약을 사용하도록 공유 타입과 브리지를 만든다.

### In Scope
- `submitPrompt`, `retryRun`, `openInWorkbench`, `moveTaskStatus`, `updateSettings` 같은 command 계약
- `getChatFeed`, `getWorkbenchLayout`, `getBoardColumns`, `getProjectDetail`, `getUsageDashboard`, `getHistoryEntry`, `getSettings` 같은 query 계약
- preload 브리지와 renderer 전용 client wrapper

### Out of Scope
- 각 command/query의 실제 데이터 처리
- DB 스키마와 외부 연동

### Acceptance Criteria
- command/query/event 타입이 renderer와 main에서 중복 정의 없이 공유된다.
- preload는 필요한 IPC surface만 노출하고 renderer에 Node API를 직접 넘기지 않는다.
- 변경 이벤트 계약은 entity 또는 projection 단위 invalidation payload를 포함한다.
- TypeScript 빌드가 계약 불일치 없이 통과한다.

### Validation Steps
- 타입 테스트 또는 빌드로 main, preload, renderer 사이 계약 일치를 검증한다.
- preload 표면을 검사해 허용된 API만 노출되는지 확인한다.
- 샘플 query 호출과 event subscription 스모크 테스트를 수행한다.

### Affected Files or Modules
- `packages/domain`
- `apps/desktop/src/preload`
- `apps/desktop/src/main/ipc`
- `apps/desktop/src/renderer/lib/ipc`

### Deferred Follow-ups
- IPC 호출 tracing과 개발용 inspector

## Story 1.5 - Event broadcast and renderer query invalidation baseline
Story Key: story-1.5
Epic Key: epic-1
Required for Success: yes
Depends On: story-1.3

### Goal
- main process commit 이후 renderer query가 같은 source of truth를 다시 읽도록 변경 브로드캐스트 기반을 만든다.

### In Scope
- transaction commit 후 domain/projection 변경 이벤트 발행
- renderer query client invalidation wiring
- 성공 commit과 실패 rollback을 구분하는 event dispatch 규칙

### Out of Scope
- 각 화면의 상세 projection 구현
- 오프라인 멀티디바이스 sync

### Acceptance Criteria
- main process의 write 성공 후 관련 query key invalidation 이벤트가 발행된다.
- 실패한 트랜잭션에서는 성공 invalidation 이벤트가 발행되지 않는다.
- renderer는 event 수신 후 screen-local mutation 대신 재조회 경로를 사용한다.
- chat feed 기준 샘플 write 후 UI 데이터가 새 조회로 갱신되는 스모크 플로우가 있다.

### Validation Steps
- 샘플 repository write 후 renderer query 캐시가 invalidate 되는지 확인한다.
- rollback 시 UI가 잘못된 성공 상태로 갱신되지 않는지 확인한다.
- main/preload/renderer 통합 테스트로 event 흐름을 검증한다.

### Affected Files or Modules
- `apps/desktop/src/main/ipc`
- `apps/desktop/src/main/events`
- `apps/desktop/src/renderer/lib/query`
- `packages/domain`

## Story 2.1 - Chat-first inbox surface and composer shell
Story Key: story-2.1
Epic Key: epic-2
Required for Success: yes
Depends On: story-1.1, story-1.2

### Goal
- 첫 화면에서 제품 목적과 첫 요청 행동이 바로 보이는 채팅 중심 인박스를 만든다.

### In Scope
- 환영 메시지, 추천 작업 카드, 한국어 입력 composer shell
- 모델 선택 UI와 최적화 모드 선택 UI
- 첫 진입 empty state와 기본 CTA copy

### Out of Scope
- 실제 prompt 저장과 실행
- run progress, 응답 렌더링, dashboard/history 화면

### Acceptance Criteria
- `Chat` 기본 화면에 한국어 환영 메시지와 첫 행동 안내가 보인다.
- 추천 작업 카드는 사업계획서 초안, 긴 PDF 요약, 카피 다듬기 같은 예시를 제공한다.
- 한국어 멀티라인 입력 composer와 모델/최적화 모드 선택 UI가 같은 작업 표면 안에 배치된다.
- empty state 문구는 사용자가 무엇을 먼저 해야 하는지 설명한다.

### Design Acceptance Criteria
- [Design] 첫 화면 실루엣은 좌측 내비게이션 + 인박스형 작업 영역 + 즉시 쓸 수 있는 composer로 읽혀야 하며, 중앙 hero 카드나 마케팅 랜딩처럼 보이면 안 된다.
- [Design] 추천 작업 카드는 일반 SaaS 카드 대시보드가 아니라 채팅 시작 행동을 보조하는 inbox starter 역할을 해야 한다.
- [Design] 기본 팔레트는 화이트 기반에 파란 primary, 민트 savings accent를 사용하되 차가운 대시보드 톤을 피한다.
- [Design] 기본 CTA는 `Submit` 같은 일반 용어가 아니라 한국어 요청 시작 맥락의 문구를 사용한다.

### Validation Steps
- 첫 화면만 보고 제품 목적과 첫 행동을 사용자 테스트 문장으로 설명할 수 있는지 확인한다.
- 추천 작업 카드 클릭이 composer 초안 채우기 또는 입력 시작 행동으로 이어지는지 확인한다.
- desktop width에서 inbox silhouette가 유지되고 모바일식 중앙 단일 카드처럼 무너지지 않는지 확인한다.

### Affected Files or Modules
- `apps/desktop/src/renderer/routes/chat`
- `apps/desktop/src/renderer/components/navigation`
- `packages/ui-shared`

### Deferred Follow-ups
- 개인화된 추천 작업 카드

## Story 2.2 - Durable Korean prompt submission and chat history preservation
Story Key: story-2.2
Epic Key: epic-2
Required for Success: yes
Depends On: story-1.3, story-1.5, story-2.1

### Goal
- 사용자가 한국어 프롬프트를 전송하면 외부 실행 전에 durable local state로 남고 채팅 히스토리에 보이게 만든다.

### In Scope
- composer submit command 연결
- 새 task/conversation/message/run 생성과 queued stage 저장
- 한국어 원문 히스토리 표시와 장문 입력 보존

### Out of Scope
- 로컬 최적화, 클라우드 호출, 응답 복원
- 요청별 절감 메타 표시

### Acceptance Criteria
- 한국어 프롬프트 전송 시 task, conversation, message, run record가 로컬에 저장된 뒤 UI에 반영된다.
- 저장된 `content_ko`는 사용자가 입력한 원문과 동일하게 유지된다.
- 긴 한국어 문장과 문서 요약 요청도 길이 손실 없이 저장되고 재실행 후 다시 보인다.
- local write 실패 시 사용자에게 안전한 오류를 보여주고 draft 입력은 사라지지 않는다.

### Design Acceptance Criteria
- [Design] 빈 상태에서 첫 전송 후 화면은 랜딩 느낌에서 대화 피드 느낌으로 자연스럽게 전환되어야 한다.
- [Design] 로딩 전 초기 상태도 사용자가 작업이 저장되었다는 신뢰를 느끼게 해야 하며, 입력창이 갑자기 초기화된 것처럼 보이면 안 된다.

### Validation Steps
- 한국어 장문 프롬프트를 전송하고 DB와 UI에서 원문이 동일한지 확인한다.
- 앱을 재실행해 같은 대화 히스토리가 복원되는지 확인한다.
- 저장 실패를 주입해 draft와 오류 메시지의 동작을 검증한다.

### Affected Files or Modules
- `apps/desktop/src/renderer/routes/chat`
- `apps/desktop/src/main/ipc/chat`
- `packages/persistence`
- `packages/workflows`

### Deferred Follow-ups
- 드래프트 자동 저장 세분화

Story Key: story-3.2
Epic Key: epic-3
Required for Success: yes
Depends On: story-2.2, story-3.1

### Goal
- 저장된 한국어 요청을 로컬 최적화 단계로 실행하고 optimized prompt artifact를 남긴다.
