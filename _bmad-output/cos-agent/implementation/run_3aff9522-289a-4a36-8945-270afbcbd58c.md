## Implementation Summary

- Added a dedicated main-process cloud gateway in `apps/desktop/src/main/providers/service.ts` with a unified `infer()` contract that only accepts `optimizedEnglishPrompt`, resolves the selected or default model, and returns either structured English output or a normalized failure payload.
- Added the initial OpenAI-backed adapter for the `gpt-4.1` path. The adapter builds a provider request from optimized English only, parses OpenAI chat-completions responses, and surfaces provider-reported token usage when it is available.
- Normalized provider failures into Korean-first recovery payloads with `code`, `message`, `guidance`, `retryable`, and optional `status`, covering `auth`, `rate_limit`, `network`, `provider_unavailable`, and `invalid_request`.
- Kept the provider layer main-process-only by rejecting renderer-context gateway creation and by leaving the preload/renderer bridge unchanged.
- Tightened the existing `story-3.2` optional cloud-dispatch seam in `apps/desktop/src/main/workflows/optimization-stage.ts` so downstream cloud handoff payloads no longer include raw Korean prompt content or conversation-summary carryover.
- Added `story-3.3` validation coverage in `apps/desktop/tests/cloud-inference-gateway.test.ts` and script aliases in `apps/desktop/package.json` plus the workspace root `package.json`.
