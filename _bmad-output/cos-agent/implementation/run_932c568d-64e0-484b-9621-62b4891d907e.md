# Implementation Summary

- Added a persisted main-process usage dashboard service in `apps/desktop/src/main/usage/service.ts` and wired it through `apps/desktop/src/main/index.ts` + `apps/desktop/src/main/ipc/register-ipc.ts`, so the real `getUsageDashboard` query now computes monthly and all-time totals from `usage_records`, `run_records`, and `tasks.usage_category` while preserving `provider`, `model`, and `pricing_version` as the comparison basis.
- Expanded the shared dashboard contract in `apps/desktop/src/shared/ipc/contracts.ts` to carry pricing-basis metadata in addition to request count, token reduction, same-period `without optimization` / `with optimization` comparison, and persisted category breakdown metrics.
- Replaced the placeholder `apps/desktop/src/renderer/routes/UsageRoute.tsx` with a query-driven screen backed by `apps/desktop/src/renderer/routes/usage-surface.ts`, including loading, error, empty, preview, monthly summary, cumulative savings, comparison, category breakdown, explicit mixed-basis states, and a separate current-month zero state so the UI never presents a merged savings proof or confuses a quiet month with mixed pricing data.
- Rewrote the Usage screen’s user-facing copy into a Korean-first surface by removing English scaffold labels and internal persistence terminology, splitting mixed-basis messaging by range so the monthly proof talks about `이번 달 기록` while the cumulative archive talks about `누적 기록`, and making the cumulative panel intro adapt to quiet-month and archive-mixed states instead of over-promising a proof that is not currently shown.
- Added a pure top-level surface-state selector so the Usage route now chooses exactly one of loading, error, empty, or content states even when the monthly and all-time queries are split between `loading` and `error`.
- Localized the visible shared shell and navigation copy in `apps/desktop/src/renderer/app/AppShell.tsx` and `apps/desktop/src/renderer/app/navigation.ts`, and removed the last English `tokens` labels from category metrics so the actual delivered Usage screen reads consistently as a Korean-first product surface instead of a scaffold shell plus a localized inner panel.
- Added usage-dashboard styling in `apps/desktop/src/renderer/styles.css` to keep the screen white-first, structurally blue, and mint-accented while using a comparison-led layout rather than a generic admin grid.
- Added automated coverage in `apps/desktop/tests/usage-dashboard.test.ts` for seeded monthly math, mixed-basis suppression, prior-month-only month-empty behavior, all-time/cumulative renderer evidence, empty-state output, persisted category alignment, Korean-first copy assertions across the shared shell plus Usage route, and source/style design checks.
