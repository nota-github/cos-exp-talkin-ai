- Extended `DesktopInvalidationEvent.source` in `apps/desktop/src/shared/ipc/contracts.ts` so renderer clients can distinguish command-driven invalidations from workflow-driven ones.
- Wired workflow invalidation into:
  - `apps/desktop/src/main/workflows/optimization-stage.ts`

- Added story-specific regression coverage in:
