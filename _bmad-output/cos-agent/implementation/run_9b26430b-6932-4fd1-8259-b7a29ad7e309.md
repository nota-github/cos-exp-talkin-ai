## User Flows or Core Requirements

- Main process and renderer must speak the same IPC vocabulary without duplicate type definitions.
- Commands remain explicit write intents instead of ad-hoc channel strings.
- Queries remain explicit read models so later screens can refetch from the same source of truth.
- Change broadcasts must carry enough entity or projection detail for renderer invalidation wiring in later stories.

## Implementation Summary

- Added a shared IPC contract source in `apps/desktop/src/shared/ipc/contracts.ts` covering the approved commands, queries, event channel, payload models, and preload/renderer API types.

- Wired Electron startup to register the shared IPC handlers in `apps/desktop/src/main/index.ts`.
- Replaced the placeholder preload surface with a typed bridge factory in `apps/desktop/src/preload/bridge.ts` and exposed it from `apps/desktop/src/preload/index.ts`.

- Added contract alignment, preload surface, and query/event smoke tests in `apps/desktop/tests/ipc-contracts.test.ts`.
- Added a small Node test-loader registration helper so `node --experimental-strip-types` can follow the TS source graph during validation.
