## Implementation Summary

- Added desktop migration `v4` in `apps/desktop/src/main/persistence/database.ts` for `projects`, `workbench_layouts`, `workbench_panels`, and `file_assets`, plus task/project/workbench lookup indexes.
- Extended the persistence contract in `apps/desktop/src/main/persistence/types.ts` and `apps/desktop/src/main/persistence/index.ts` with project, workbench, board, recent-task, and file-asset records and repositories.
- Expanded `apps/desktop/src/main/persistence/repositories.ts` with:
  - canonical task workflow updates via `tasks.updateWorkflow(...)`
  - recent-task rail projection via `tasks.listRecent(...)`
  - board projection via `board.getColumns()`
  - project list/detail queries via `projects.list()` and `projects.getDetail(...)`
  - workbench layout/panel persistence via `workbenchLayouts.*` and `workbenchPanels.save(...)`
  - file-asset metadata CRUD via `fileAssets.*`
- Kept `task.status` and `task.project_id` as the only workflow/project truth. The new tables reference `task_id` or `project_id` but do not duplicate task status.
- Added `apps/desktop/tests/work-management-persistence.test.ts` for the story acceptance path and updated legacy migration/version assertions in `apps/desktop/tests/persistence.test.ts` and `apps/desktop/tests/settings-keychain.test.ts`.
