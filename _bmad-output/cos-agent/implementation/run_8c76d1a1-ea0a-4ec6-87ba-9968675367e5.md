## Implementation Summary

- Added a persisted main-process history inspection service in `apps/desktop/src/main/history/service.ts` and wired it through `apps/desktop/src/main/index.ts` and `apps/desktop/src/main/ipc/register-ipc.ts`, so the app can now read a run-history list and one run-detail view directly from stored messages, prompt artifacts, and usage rows.
- Expanded the desktop IPC contract in `apps/desktop/src/shared/ipc/contracts.ts` with `getHistoryFeed`, richer `HistoryEntryResult` evidence types, and matching preload/query-cache support in `apps/desktop/src/preload/bridge.ts` and `apps/desktop/src/renderer/lib/ipc/query-client.ts`.
- Added `apps/desktop/src/renderer/routes/history-surface.ts` as the pure renderer helper for copy, preview data, usage cards, and reveal rules, then extended `apps/desktop/src/renderer/routes/UsageRoute.tsx` with a run inbox rail plus an evidence detail panel that keeps final Korean response and source Korean visible first while hiding optimized English and provider English artifacts behind explicit reveal actions.
- Added dedicated history inspection styles in `apps/desktop/src/renderer/styles.css` so the Usage / History area reads as a desktop workbench surface rather than a raw debug table.
- Added `apps/desktop/tests/history-inspection.test.ts` to verify persisted artifact fidelity, collapsed/expanded advanced sections, and a final-response-first list experience.
