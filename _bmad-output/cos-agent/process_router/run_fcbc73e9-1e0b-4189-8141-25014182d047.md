# Process Router Decision

## Route Summary
Talkin AI is not a small single-flow prototype. The product scope combines a local LLM optimization loop, multiple cloud-model integrations, API key handling, persisted desktop workspace state, token/cost analytics, and several connected user-facing surfaces such as chat, workbench, kanban, projects, history, and settings.

The approved brief and success criteria require stronger decomposition and evidence than `plan_lite` or `standard_planning` are designed to guarantee. The recommended depth signal is `rigorous_build`, and the project contains concrete integration and persistence risks that justify the matching fixed planning contract.

## Selected Planning Contract
`rigorous_planning`

This is the best fit because:

- The product depends on external integrations across local optimization infrastructure and cloud AI providers.
- The app must store and protect API keys, conversation state, project context, and cost history.
- The workflow spans multiple synchronized desktop surfaces rather than one narrow end-to-end flow.
- The brief still leaves implementation-critical uncertainty around provider rollout order, local LLM choice, and savings calculation method.

`plan_lite` is too shallow for this scope, and `standard_planning` would under-specify the integration, persistence, and evidence burden for the requested experience.

## Risk Flags
- `external_integration`
- `sensitive_data`
- `broad_workflow`
- `ambiguous_scope`

## Guardrail Explanation
This route choice increases planning and validation rigor, but it does not imply operational readiness. The next phases should keep the required PRD, architecture, stories, stronger validation/evidence expectations, and release-readiness notes while keeping those release-readiness notes separate from any claim that the product is production-ready.

This phase does not create PRD, architecture, stories, or implementation code. It only selects the safest fixed planning contract and records the guardrails that later phases must preserve.
